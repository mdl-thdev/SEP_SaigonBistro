Saigon Bistro is an interactive web-based customer service Help Page designed to handle
customer inquiries, issues, and support tickets efficiently.

The application simulates a real-world support workflow where customers can submit help
requests and administrators can manage, assign, and resolve tickets through an admin dashboard.

## Design Process

### Project Purpose

Provide a central help desk for Saigon Bistro customers
Allow ticket submission, tracking, and resolution
Demonstrate API-driven support workflows
Showcase admin ticket assignment and lifecycle management

### Target Users

Customers needing help with orders, delivery, or accounts
Admin / Support staff managing incoming tickets

## Features

### Existing Features

- Feature 1: Customer can submit support tickets via Help page
- Feature 2: Customer can view ticket status (Open / In Progress / Closed)
- Feature 3: Customer can secure access using login credentials
- Feature 4: Admin can view all submitted tickets
- Feature 5: Admin can assign tickets to support staff
- Feature 6: Admin and staff can update ticket status
- Feature 7: Admin and staff can track ticket history and actions

##  Technologies Used

### Frontend

- **HTML5**  
  https://developer.mozilla.org/en-US/docs/Web/HTML  
  HTML5 was used to structure the content of the web application and define the layout of pages.

- **Tailwind CSS**  
  https://tailwindcss.com  
  Tailwind CSS was used to style the application efficiently using utility-first classes, enabling a clean, responsive, and consistent design.

- **Vanilla JavaScript**  
  https://developer.mozilla.org/en-US/docs/Web/JavaScript  
  JavaScript was used to implement front-end interactivity, handle user actions, and manipulate the DOM for dynamic content updates, and communicate with backend APIs.


### Backend

- **Node.js**  
  https://nodejs.org  
  Node.js was used as the runtime environment to execute server-side JavaScript and build the backend logic.

- **Express.js**  
  https://expressjs.com  
  Express.js was used to create RESTful APIs, handle HTTP requests, and serve static frontend files.

- **bcryptjs**  
  https://www.npmjs.com/package/bcryptjs  
  bcryptjs is used to securely hash user passwords before storage. This ensures that plaintext passwords are never stored and improves overall authentication security. Passwords are verified by comparing hashed values during login.

### Data Storage

- **Supabase**  
  https://supabase.com
  Supabase is used as the primary backend service for data storage and management. It provides a hosted PostgreSQL database along with authentication and API access.
